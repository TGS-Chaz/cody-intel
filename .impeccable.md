# Impeccable Design Context — cody-intel

## Design Context

### Users

Same audience family as cody-crm, but in a different mode:

- **Cannabis business owners / operators** (primary). At a desk, longer analytical sessions. Reading maps, comparing stores, tracking brand distribution, digesting regulatory news. Session length: 5–30 minutes. Primary need: **pattern recognition** across dense data.
- **Sales reps and managers** referencing Intel between CRM sessions — pulling up a dispensary's market position before a visit, checking what competitors carry, validating a territory call. Secondary need: **quick lookup inside a reference-grade dataset**.
- **Farm operators** tracking where their SKUs are placed, which stores are growing, where they're losing shelf space.

Unlike cody-crm (30–90 second task-entry loops), Intel sessions are analytical. Users lean in, scroll, zoom maps, compare charts. They expect dense, trustworthy data — not a marketing dashboard.

### Brand Personality

**Intelligence-grade. Analytical. Premium.**

Shared identity with cody-crm: cannabis industry weapon, unfair advantage. For Intel specifically, emphasize the "intelligence" half of the triad — this should feel like a Bloomberg Terminal for cannabis, not a consumer analytics product. Quiet, confident, data-trusting.

References to emulate:
- **Bloomberg Terminal** — density, trust, zero decoration, every pixel earns its place.
- **Stripe dashboards** — beautiful data at rest, premium, readable at a glance.
- **Linear** — restraint, keyboard ergonomics, clean information architecture.
- **Kepler.gl / Mapbox Studio** — map craft when it matters.

Anti-references:
- Tableau / Power BI — bloated, dated, enterprise-sludge.
- Google Analytics 4 — confusing, over-designed, hostile to power users.
- Generic SaaS analytics templates (single big KPI + sparkline x 4).

### Aesthetic Direction

- **Dark mode is the hero.** Analytical work is often done in dim offices or evenings; dark mode reduces eye fatigue during long sessions. Light mode must still be polished.
- **Teal stays as primary brand identity** (`#00B894` light / `#00D4AA` dark). Shared with cody-crm — these products are visually recognizable as siblings.
- **Orange reserved for CTA / conversion** — rarer here than in CRM since Intel is mostly read-only analysis. Use it only for primary actions (save view, share report).
- **Data-first visual language.** Charts, maps, and tables are primary citizens — not decorative cards wrapped around numbers. Typography and color serve the data. Grid lines, axis labels, and legends must be crafted carefully, not left as library defaults.
- **Tinted neutrals** carrying a subtle teal cast for cohesion — same rule as cody-crm.
- **Typography:** Satoshi / General Sans shared with cody-crm. Add a high-quality monospace or tabular face for data columns, codes, coordinates — but avoid reflex picks (no IBM Plex Mono, no Space Mono, no JetBrains Mono). Consider tabular-nums on all numeric cells regardless of font.
- **Bloomberg-dense, Linear-clean.** Show a lot at once; make it scannable via hierarchy, color saturation, and typographic rhythm — not whitespace decoration.

### Constraints

- **Data visualization is primary.** Chart libraries (recharts, visx, mapbox, etc.) must be themed to match — no off-the-shelf chart aesthetics leaking through.
- **Map ergonomics:** maps are a first-class surface. Basemap style should match the app theme (dark map in dark mode, not a jarring OSM default). Cluster styling, legend, tooltips all custom.
- **Table density:** users want to see many rows. Don't pad rows for "comfort" when the user wants comfort *density*. Provide variant row heights if needed.
- **Performance with big data:** virtualized lists/tables where row counts exceed ~200. Never ship a chart that re-renders on every prop change.
- **Accessibility:** solid contrast, logical tab order, readable at arm's length. Maps need keyboard access for pan/zoom where possible; charts need text alternatives for critical insights.
- **Reduced motion honored** — analytical users don't want scroll-linked animation on their decision-making tools.

### Design Principles

1. **Data is the subject. Chrome is the frame.** Everything in the interface either displays data or helps the user interrogate it. No decoration. No hero metrics as wallpaper. No gradients over numbers.
2. **Density with hierarchy.** Bloomberg-dense, Linear-clean. Show more than a consumer product dares, but make it scannable through typography, color weight, and spacing — not card decoration.
3. **Trust through craft.** Axis labels, legends, tooltips, map styling — polish the parts users overlook. Sloppy chart defaults signal sloppy data. Pristine charts signal reliable data.
4. **Teal identity, orange action (rarely), tinted neutrals.** Same color rules as cody-crm. One accent, applied consistently; colors inside charts follow a deliberate encoded palette, not rainbow defaults.
5. **Dark mode is first-class.** Including charts and maps — not just UI chrome. Light mode is the follow-through.
6. **Earn every pixel.** Especially true for analytics: if a chart, card, or column doesn't change decisions, remove it.
7. **Cohesion with cody-crm.** These products must feel like one platform. Same tokens, same typography, same motion language. A rep moving between them should not experience a seam.
